Définition de la Virtualisation

La virtualisation est une technologie qui consiste à créer une version simulée ou virtuelle d'une ressource informatique, comme un serveur, un stockage, un réseau ou un système d'exploitation. Elle permet de faire fonctionner plusieurs systèmes d'exploitation et applications sur un seul serveur physique, en isolant chaque environnement virtuel.

Différence entre Émulateur, Simulateur et Outil de Virtualisation

  • Émulateur : Reproduit fidèlement le comportement d’un matériel ou d’un logiciel sur un autre système, souvent avec des performances moindres. Exemple : émulateurs de consoles de jeux, ou encore QEMU, un émulateur connue sur Linux, compatible avec la plupart des systèmes Unix.
  • Simulateur : Recrée le fonctionnement d’un système sans nécessairement imiter le matériel sous-jacent, souvent utilisé pour le développement ou les tests. Exemple : simulateurs d’avion.
  • Outil de Virtualisation : Permet de créer et gérer des machines virtuelles qui partagent les ressources physiques d’un hôte tout en étant isolées les unes des autres. Exemple : VirtualBox ou VMware.

Type de Virtualisation pour les Environnements VirtualBox, VMware Workstation & ESXi, KVM, Stratus

  • VirtualBox et VMware Workstation : Utilisent la virtualisation de type 2 (hyperviseur de second niveau), où le logiciel de virtualisation s'exécute sur un système d'exploitation hôte.
  • VMware ESXi est de type 1.
  • KVM, sur Linux, est un hyperviseur de type 1.
  • Stratus : Peut impliquer une virtualisation de type 1 (hyperviseur de premier niveau), où le logiciel de virtualisation fonctionne directement sur le matériel, mais cela dépend de l’implémentation.

Forme/Lieu de Sauvegarde de la Machine Virtuelle Créée

Les machines virtuelles peuvent être sauvegardées sous forme de fichiers sur le disque dur de l'hôte. Ces fichiers incluent généralement :

  • Le disque virtuel (au format VDI, VMDK, QCOW2 pour QEMU/KVM etc.).
  • Les fichiers de configuration de la machine virtuelle (paramètres système et matériel).
  • Les snapshots pour les sauvegardes incrémentales.

La sauvegarde peut être stockée localement, sur un serveur de sauvegarde, ou dans le cloud selon les besoins de l'utilisateur.
